# Critical Project First Thread Example

```text
这是一个研究项目，项目路径是：
<local_project_root>

请使用 `$md-project-kickoff` skill。

Mode: design only。

请先判断哪个目录应该作为项目根目录。
然后按 Git-first 方式建立轻量项目启动结构：

1. 检查或初始化本地 Git，把本地作为 source of truth；
2. PROJECT_INDEX.md；
3. AGENTS.md；
4. docs/definitions.md；
5. docs/method_registry.md；
6. docs/codex/analysis_contract_template.md。

不要实现分析代码。

这个项目大方向是比较不同条件下某个核心现象或指标的变化机制。
请用 grill-me / 追问澄清 的方式帮我梳理：

1. 核心科学问题；
2. 已有数据和 run 的路径；
3. 必须先确认的定义，例如样本或系统标签、核心指标、事件判据、时间窗口、阈值、单位；
4. 第一个最小可执行分析应该是什么；
5. 哪些文件夹不能扫描、移动、删除或下载。

如果项目确实需要远程运行或文献综述，再单独补，不要第一轮就展开。

最后请列出“需要我确认的问题”，不要超过 5 个。
```
